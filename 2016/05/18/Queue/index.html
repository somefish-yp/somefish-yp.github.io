Queue原理分析

public interface Queue<E> extends Collection<E>

通过看Queue的签名可以直观的看到，Queue是一个接口，继承于Collection。
Queue提供了以下几个方法

boolean add(E e);  
boolean offer(E e);    
E remove();  
E poll();  
E element();  
E peek();

队列是一种特殊的线性表，实现了该接口的有LinkedList。
与单向链表不同，队列是FIFO(first-in-first-out),先进先出。

Queue<String> que = new LinkedList<String>();
    que.offer("d");
    que.offer("a");
    que.offer("r");
    que.offer("g");
    que.offer("o");
    que.offer("t");
    System.out.print("正常输出：");
    for(String str : que){
        System.out.print(str);
    }
    System.out.println();

    System.out.print("poll:");
    que.poll();//得到第一个元素并删除第一个元素
    for(String str : que){
        System.out.print(str);
    }
    System.out.println();

    System.out.print("peek:");
    que.peek();//得到第一个元素
    for(String str : que){
        System.out.print(str);
    }
    System.out.println();

    System.out.print("element:");
    que.element();//得到第一个元素
    for(String str : que){
        System.out.print(str);
    }

    正常输出：dargot
    poll:argot
    peek:argot
    element:argot

可以看到字母d先进入，然后也是先输出字母d。
